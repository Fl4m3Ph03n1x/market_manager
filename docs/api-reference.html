      Modules

      AuctionHouse

      Library representing the interface for the auction house. Responsible for making calls and decoding the answers from the auction house into a format the manager understands.

      AuctionHouse.Impl.HTTPClient

      Adapter for the interface AuctionHouse

      AuctionHouse.Impl.Settings

      Holds configurations that do not depend on the environment. 12 factor app standard states that only configurations that depend on the environment should be configurable in config files (config folder).

      AuctionHouse.Runtime.Server

      Process responsible for managing requests to the auction house. Talks to the logic layer and terminates gracefully.

      AuctionHouse.Type

      Holds the types for this library.

      Manager

      MarketManager is an application that allows you to make batch requests to warframe.market. This is the entrypoint of everything. If you have a module and you need to talk to MarketManager, this is who you call, the public API.

      Manager.Impl.Interpreter

      Core of the manager, where all the logic and communication with outer layers is. Currently, it works more like a bridge between the different ports of the application and manages data between them.

      Manager.Impl.PriceAnalyst

      Strategies calculate the optimum price for you to sell an item.

      Manager.Impl.Strategy

      Behaviour that specifies the functions any strategy must implement. It is mostly used to provide an implementation of the Strategy pattern from OO languages, but adapted to Elixir.

      Manager.Impl.Strategy.EqualToLowest

      Gets the lowest price for the given item and uses it.

      Manager.Impl.Strategy.LowestMinusOne

      Gets the lowest price for the given item and beats it by 1.

      Manager.Impl.Strategy.TopFiveAverage

      Gets the 5 lowest prices for the given item and calculates the average.

      Manager.Impl.Strategy.TopThreeAverage

      Gets the 3 lowest prices for the given item and calculates the average.

      Manager.Runtime.Server

      Process responsible for taking requests from the interface and directing them to the appropriate layers. Supervises dependencies to make sure they are regenerated should something fail.

      Manager.Runtime.Worker

      Process responsible for doing asynchronous tasks. The Worker is supervised by the Server and is restarted should something fail. It communicates with the calling client via send/2, since we don't know if the client is a GenServer or not. Should this change (and we are sure the client is a GenServer) then GenServer.cast should be used instead.

      Manager.Type

      Holds the types for the Manager library.

      Shared

      Holds general utility functions used across all the project, as well as the DataStructures and Types used to represent the Domain.

      Shared.Data.Authorization

      Saves authorization details for a user.

      Shared.Data.Credentials

      Represents the credentials of an user. Credentials are used to get access to Authorization and User info, which is the used to makes requests to the market.

      Shared.Data.Order

      Represents an order to be made to warframe market. There are two types of orders thus far, RankedOrders and SimpleOrders. See respective files for more information.

      Shared.Data.Order.RankedOrder

      Represents an order to be made to warframe market with a rank.

      Shared.Data.Order.SimpleOrder

      Represents an order to be made to warframe market without a rank.

      Shared.Data.OrderInfo

      Represents information about current orders from warframe market.

      Shared.Data.OrderInfo.User

      Represents the account information for a warframe.market User who has an order posted.

      Shared.Data.PlacedOrder

      Represents an order that was placed by the User of this application. I could have used OrderInfo as well, but since the later one has a ton of information I don't really need/want to deal with, I opted for this instead.

      Shared.Data.Product

      A Product is an item we can create a sell order for. Currently we only support mods. Products contain warframe.market information about the item, as well as default parameters to create sell orders.

      Shared.Data.Strategy

      A strategy for selling a product.

      Shared.Data.Syndicate

      A syndicate is a party with tradable items.

      Shared.Data.User

      Represents relevant User information for clients using this AuctionHouse.

      Shared.Utils.ExtraGuards

      Contains additional guards to use in functions.

      Shared.Utils.Maps

      Set of utility functions used across the app to help with Map operations.

      Shared.Utils.Structs

      Set of utility functions used across the app to help with Struct operations.

      Shared.Utils.Tuples

      Set of utility functions used across the app to help with Tuple operations.

      Store

      Port for the persistency layer. This module contains the Public API for the storage app.

      Store.FileSystem

      Adapter for the Store port, implements it using the file system.

      Store.Type

      Contains the types for the Store library. Because we separate this file from the API and implementation, we also avoid a cyclical dependency between them.

      WebInterface

      The entrypoint for defining your web interface, such as controllers, components, channels, and so on.

      WebInterface.ActivateLive
      WebInterface.Components.OperationProgress

      LiveView subcomponent for the Operation Progress Page. It hides both sidebar and main subcomponents and displays information about operations taking place.

      WebInterface.CoreComponents

      Provides core UI components.

      WebInterface.DeactivateLive
      WebInterface.Desktop.MenuBar

      Menubar that is shown as part of the main Window on Windows/Linux. In MacOS this Menubar appears at the very top of the screen.

      WebInterface.Desktop.WindowUtils

      Contains utility functions to determine window attributes.

      WebInterface.ErrorHTML
      WebInterface.ErrorJSON
      WebInterface.LoginLive
      WebInterface.LogoutLive
      WebInterface.PageController
      WebInterface.PageHTML
      WebInterface.Persistence

      Responsible for temporary persistence in WebInterface. Saves state that needs to be shared between LiveViews. Uses ETS beneath the scenes.

      WebInterface.Persistence.Button

      Persistence module to know which buttons are selected.

      WebInterface.Persistence.Strategy

      Persistence module for strategy data. Mostly to track which strategy is selected in the menu and access the list of strategies.

      WebInterface.Persistence.Syndicate

      Persistence module for syndicate data. Mostly to track selected options in the menu and access the list of syndicates.

      WebInterface.Persistence.User

      Persistence module for user related data.
